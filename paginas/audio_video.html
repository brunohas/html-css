ÁUDIO

Para reproduzir um áudio MP3 em um site, utilizamos a tag <audio>, irá abrir um parâmetro src=" " (colocar o nome do arquivo) e também é preciso colocar os parâmetros autoplay para tocar a música assim que carregar a página (não é obrigatório), controls para aparecer um "controle" para dar play/pause e aumentar ou diminuir o volume do áudio, loop para o áudio se repetir (usa-se normalmete quando o áudio é curto). 


    <audio src="midia/THE BOX MEDLEY FUNK 2.mp3" controls></audio>

Também existe outros tipos de formatos de áudio (WAV e OGG), ver se tem compatibilidade com o navegador. Outra maneira de reproduzir áudios também é com a tag <audio>, porém não é necessário o parâmetro que já vem incluso (pode apagar o src=" ") se quiser pode acrescentar o parâmetro preload=" " (auto = só considera que a página carregou depois de carregar todo o áudio (não recomendado) / metadata = só carrega o nome, data, duração do arquivo / none = só carrega quando o usuário apertar play no controle). Depois colocar a tag <source:src> dentro da tag <audio>, assim abrindo outro parâmetro type=" ", para áudios em MP3 colocar o type="audio/mpeg" / para áudios em WAV colocar o type="audio/wav" / para áudios em OGG colocar o type="audio/ogg".

Exemplo:


    <audio controls>
        <source src="midia/THE BOX MEDLEY FUNK 2.mp3" type="audio/mpeg">  !- primeiro áudio a tentar carregar -!
        <source src="midia/THE BOX MEDLEY FUNK 2.wav" type="audio/wav">   !- se não carregar o primeiro, carrega o segundo -!
        <source src="midia/THE BOX MEDLEY FUNK 2.ogg" type="audio/ogg">   !- se não carregar o segundo, carrega o terceiro -!
        <p>
            Infelizmente seu navegador não consegue reproduzir áudio.     !- Só irá mostrar se não conseguir produzir o áudio -!
        </p>
    </audio>

VÍDEO

Os formatos mais utilizados em vídeos são o MP4 e M4V, existem outros como WebM (que é um arquivo bem pequeno, porém não tem muita qualidade de vídeo) e OGV (um formato que está intimamente ligado ao OGG, pode ser utilizado para armazenamento ou transmissão na web). Para reproduzir um vídeo utilizamos a tag <video>, porém conseguimos fazer que nem no áudio, conseguimos colocar mais de um tipo de arquivo utilizando a tag <video> e adicionando a tag <source:src> para cada tipo de arquivo. Depois colocar de colocar a tag <source:src> dentro da tag <audio>, assim abrindo outro parâmetro type=" ", para vídeos em MP4 colocar o type="video/mp4" / para vídeos em WebM colocar o type="video/webm" / para vídeos em OGV colocar o type="video/ogv".

Para pesquisar vídeos, podemos também utilizar o site Pexels que é o mesmo para pesquisar imagens (sempre lembrar dos direitos autorais). Para converter/formatar um vídeo utilizamos o aplicativo HandBrake (só converte para MP4, M4V e WebM / para converter para OGV basta pesquisar no google "conversor ogv"), serve para converter o vídeo em algum outro tipo de arquivo para que o vídeo fique mais leve, ou para o vídeo ficar com uma resolução melhor. Não é indicado hospedar um arquivo muito no grande no site, pois o custo fica muito caro. Se for colocar algum arquivo muito pesado, é indicado colocar ele como um link externo.

Vídeos hospedados localmente (dentro do servidor)

Para reproduzir vídeo em um site, utilizamos a tag <video> (se o vídeo não couber na tela, você pode editar/formatar ou adicionar um parâmetro width: " " e colocar em quantos pixels de largura o vídeo vai ficar), colocar parâmetro controls para mostrar um controle para dar play/pause ou aumentar/diminuir o volume. Vídeos hospedados localmente, consomem muitos dados do servidor, assim ficando mais caro para hospedar.
Ex: 


    <video controls>
        <source src="midia/video-drift-600.mp4" type="video/mp4">    !- primeiro vídeo a tentar carregar -!
        <source src="midia/video-drift-600.webm" type="video/webm">  !- se não carregar o primeiro, carrega o segundo -!
        <source src="midia/video-drift-600.ogv" type="video/ogv">    !- se não carregar o segundo, carrega o terceiro -!
        <p>
        Infelizmente seu navegador não consegue reproduzir vídeo. !-só irá mostrar se não conseguir reproduzir nenhum vídeo-!
        </p>
    </video>

Vídeos hospedados externamente

Os vídeos hospedados externamente tem como "benefício" não utilizar os dados do servidor para carregar o vídeo, assim ficando mais barato para hospedar. Para inserir um vídeo externo, copiamos o <iframe> gerado pelo site que contém o vídeo.

  • Vídeos do Youtube → Para conseguir inserir um vídeo do Youtube no site, basta clicar em compartilhar no vídeo desejado e clicar em Incorporar (monta um <iframe>), depois só copiar e colar o código (conseguimos mudar o tamanho para ser exibido, basta alterar o width: e o heigth:).
  • Vídeos do Vímeo → Para conseguir inserir um vídeo do Vimeo no site, basícamente é igual a do Youtube, porém o Vímeo é um aplicativo pago. (Não coloquei nenhum vídeo pois não tenho conta).

LINKS

Links externos

Para utilizar links externos usar a tag <a>, assim abrindo um parâmetro href=" " para colocar o link desejado. Depois colocamos mais 2 parâmetros: rel="external" (identificar que é um link externo) ou rel="nofollow" (avisa o mecânismo de busca que você não da o aval do conteúdo do link), e target="_self" (abrir o link sobre a página) ou target="_blank" (abrir o link em uma nova janela).

Links internos

Para links internos também utiliza-se a tag <a>, abrindo o parâmetro href=" ", porém colocamos o nome do arquivo HTML no lugar (ex: index.html). Depois colocamos mais 2 parâmetros: rel="next" (próxima página) ou rel="prev" (página anterior), e target="_self" (abrir o link sobre a página) ou target="_blank" (abrir o link em uma nova janela).

Nas 2 opções conseguimos adicionar uma tag de estilo cursor: para mudar o cursor do mouse.

  • wait → carregando
  • not-allowed → "proibido"
  • grab → pegar
  • pointer → "mãozinha"

IFRAME

Um iframe é uma etiqueta HTML que permite a incorporação de outro documento HTML dentro de uma página, ele pode ser usado em qualquer lugar da página, sem precisar incluí-lo na estrutura do layout do site. O iframe é usado para:

  • Exibir vídeos, mapas, anúncios e feeds de mídia social
  • Integrar conteúdo interativo
  • Carregar páginas externas dentro da página web
  • Visualizar conteúdos de terceiros
  • Adicionar um vídeo do Youtube em post
  • Inserir mapas do Google Maps em sites institucionais

Para modificar o comportamento do iframe, é possivel usar atributos como:

  • width: e heigth: para definir a largura e altura do elemento (padrão é width: 300px; heigth: 150px;)
  • sandbox: para aplicar camada extra de segurança
  • ::-webkit-scrollbar {width: 0px; heigth: 0px;} para eliminar barra de rolagem

Utilizar o iframe usando a tag <iframe src=" " frameborder="0">, o src=" " é o direcionamento do link e o frameborder=" " é a borda da janela (0 = sem borda / 1 = com borda). Se atentar pois tem sites que tem uma proteção contra o iframe. E também existem dispositivos que não tem o suporte para o iframe, nesse caso, colocar uma âncora <a> dentro da tag <iframe>, para que se o iframe não abrir, tenha a âncora para direcionar a pessoa para o site, exemplo: 


<iframe src="https://www.youtube.com/" frameborder="0"><a href="https://www.youtube.com/" target="_blank">Youtube</a></iframe>

Ainda dentro da tag <iframe> conseguimos acrescentar o parâmetro scolling="auto" (se o conteúdo ultrapassar o tamanho do iframe, ele vai criar uma barra de rolagem), scolling="yes" (mesmo que o conteúdo caiba no iframe, irá aparecer a barra de rolagem), ou scolling="no" (mesmo que o conteúdo seja maior, ele não vai permitir a barra de rolagem).

Segurança do iframe

Se atentar na hora de colocar iframe no seu site, pois tem sites que capturam informações do usuário. Para bloquear essas possíveis ações, utilizar o parâmetro sandbox="sandbox" (é utilizado para isolar conteúdos não confiáveis em um iframe). Outra forma de deixar seu site mais seguro, utilizar o parâmetro referrerpolicy=" " (é uma ferramenta que controla a quantidade de informações de referência que os navegadores transmitem), o padrão é referrerpolicy="no-referrer-when-downgrade" (remove a referência ao fazer downgrade de protocolo, como de HTTPS para HTTP), se quiser tornar o que está carregado no iframe um pouco mais limitado, utilizar o parâmetro referrerpolicy="no-referrer" (o conteúdo que está carregado no iframe não vai coletar nenhuma informação do usuário). Então para uma segurança forte do iframe, utilizar os parâmetros sandbox="sandbox" + referrerpolicy="no-referrer" dentro da tag do iframe. Se o iframe for da mesma origem conseguimos desligar algumas configurações, utilizamos também o parâmetro sandbox=" ", agora vou mostrar algumas configurações de sandbox:

  • sandbox="allow-same-origin" → permite que o conteúdo seja tratado sendo da mesma origem.
  • sandbox="allow-forms" → permite formulários.
  • sandbox="allow-scripts" → permite scripts.
  • sandbox="allow-orientation-lock" → permite bloquear a orientação da tela.
  • sandbox="allow-pointer-lock" → reativa as APIs de bloqueio dentro do iframe.
  • sandbox="allow-popups" → permite pop-ups.
  • sandbox="allow-popups-to-escape-sandbox" → permite que pop-ups abram novas janelas sem herdar o sandboxing.
  • sandbox="allow-top-navigation" → permite que o conteúdo navegue em seu contexto de navegação de nível superior.
  • sandbox="no-values" → aplica todas as restições, desativando a maioria das funcionalidades do conteúdo incorporado.
